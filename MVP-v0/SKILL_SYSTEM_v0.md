# Snapell 技能系統 — v0 版（僅定義，不結算）

> **v0 定位：** 技能在 v0 階段只是「卡面上顯示的文字和數值」，不進行對戰結算。
> 但數值必須正確計算並存入 Firestore，v1 對戰時直接使用。
>
> **完整設計參考：** `MVP/SKILL_SYSTEM_MVP.md`

---

## 1. v0 技能範圍

| | 完整 MVP | v0 |
|---|---------|-----|
| 技能總數 | 13（7 共享 + 6 專屬） | **7（共享）** |
| 專屬技能 | spite, momentum, leech, hex, final_stand, rebirth | **延後到 v1** |
| 結算邏輯 | Phase 1~4 完整結算 | **不結算** |
| 數值計算 | ✅ | ✅（存入 Firestore 供 v1 使用） |
| AI 動態命名 | ✅ | ✅ |

### 為什麼 v0 不做專屬技能

```
1. v0 不做對戰 → 技能只是展示文字 → 7 種共享已經夠豐富
2. 專屬技能的身份感在「對戰中觸發」時才有意義
   → spite 死後反傷、rebirth 復活 — 這些只在對戰中才有戲劇性
3. v0 的🟡傳說和🔴神話使用共享技能
   → v1 時新生成的🟡🔴才有專屬技能
   → v0 舊卡保留原技能不變（不影響體驗）
```

## 2. 7 種共享技能定義

| # | 機制 ID | 效果 | 基準值（🟣） | v0 可用稀有度 |
|---|---------|------|------------|--------------|
| 1 | `atk_boost` | ATK +X% | 25% | ⚪🟢🔵🟣🟡🔴 |
| 2 | `def_boost` | DEF +X% | 35% | ⚪🟢🔵🟣🟡🔴 |
| 3 | `shield` | 受到傷害 -X% | 30% | ⚪🟢🔵🟣🟡🔴 |
| 4 | `counter` | 反彈傷害 X% | 45% | ⚪🟢🔵🟣🟡🔴 |
| 5 | `madness` | ATK ×N, DEF=0 | ×2.0 | 🟢🔵🟣🟡🔴 |
| 6 | `double_strike` | 攻擊兩次，第二擊 X% | 50% | 🔵🟣🟡🔴 |
| 7 | `pierce` | 穿透 DEF X% | 45% | 🔵🟣🟡🔴 |

> **v0 的退役規則暫時放寬。** 完整 MVP 中 atk_boost/def_boost 在🟣+退役，但 v0 沒有專屬技能來填充池子。v0 的所有共享技能在所有可用稀有度都可選（madness 仍然🟢起、double_strike/pierce 仍然🔵起）。v1 加入專屬技能後恢復退役規則。

## 5. AI 技能選擇指引（v0 版）

與完整 MVP 基本一致，差異：

- 可選列表只有 7 種共享技能（按稀有度過濾）
- 無專屬技能優先提示
- 🔴 神話不再強制 rebirth，而是從 madness/double_strike/pierce 中選

```
v0 各稀有度池大小：
  ⚪ → 4（atk_boost, def_boost, shield, counter）
  🟢 → 5（+madness）
  🔵 → 7（+double_strike, pierce）
  🟣 → 7（全部可用）
  🟡 → 7（全部可用）
  🔴 → 7（全部可用）
```

> **v0→v1 注意：** v1 恢復退役規則和專屬技能後，🟣🟡🔴的池子會縮小+加入專屬技能。

## 6. 卡面技能顯示規格

v0 技能不結算，但卡面需要正確顯示：

```
卡面技能區域：
┌─────────────────────────────┐
│ [技能icon] 肉球連擊           │
│ 揮出兩記致命肉球，第二擊+5    │
└─────────────────────────────┘
```

- `skillName`：AI 生成的獨特名稱（2~5 字繁體中文）
- `skillDesc`：包含計算後的絕對數字，不顯示百分比

## 7. v0 → v1 遷移

### 新增 6 種專屬技能

v1 回補：spite(⚪), momentum(🟢), leech(🔵), hex(🟣), final_stand(🟡), rebirth(🔴)

### 恢復退役規則

| 技能 | v0 可用 | v1 可用（恢復退役） |
|------|--------|-------------------|
| atk_boost | ⚪🟢🔵🟣🟡🔴 | ⚪🟢🔵 |
| def_boost | ⚪🟢🔵🟣🟡🔴 | ⚪🟢🔵 |
| shield | ⚪🟢🔵🟣🟡🔴 | ⚪🟢🔵🟣 |
| counter | ⚪🟢🔵🟣🟡🔴 | ⚪🟢🔵 |
| madness | 🟢🔵🟣🟡🔴 | 🟢🔵🟣🟡 |
| double_strike | 🔵🟣🟡🔴 | 🔵🟣🟡 |
| pierce | 🔵🟣🟡🔴 | 🔵🟣🟡 |

### 舊卡處理

**v0 期間已生成的卡片保留原技能，不做遷移。** 例如 v0 生成的 🔴 神話有 double_strike，v1 後它仍然保留 double_strike（而非被強制改為 rebirth）。只有 v1 之後新生成的 🔴 神話才是 rebirth。

這是可以接受的，因為：
1. v0 期間卡片數量少
2. 玩家對早期卡片有情感依附（Endowment Effect）
3. 「早期版本的特殊卡」反而有收藏價值
