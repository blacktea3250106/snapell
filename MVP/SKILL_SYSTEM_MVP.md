# Snapell 技能系統 — MVP 版

> **所屬：** MVP 製卡 + 對戰系統共用
> **完整版：** `SKILL_SYSTEM.md`（30 種技能）
> **MVP 原則：** 砍掉互動邊界複雜的技能，保留每個稀有度的核心身份感。v2 再展開完整 30 技能池。

## 1. 設計總覽

### 核心規則

- **13 種技能機制**（7 共享 + 6 稀有度專屬）
- **每個稀有度 1 個專屬技能**：作為該稀有度的身份標誌
- **階梯式解鎖 + 退役**：隨稀有度提升，解鎖新技能，淘汰基礎技能
- **全被動**：翻牌時自動觸發，不需要玩家操作
- **百分比後端**：技能效果以百分比定義，系統計算後在卡面顯示絕對數字
- **AI 動態命名**：同一種技能機制，不同卡片擁有不同的技能名稱和描述

### 設計哲學：技能隨稀有度「變有趣」

```
⚪ 凡品：基礎功能 + 專屬 spite（死後反傷）→ 白卡也有存在感
🟢 精良：開始有花樣 + 專屬 momentum（越打越強）→ 條件觸發入門
🔵 稀有：全面展開 + 專屬 leech（跨回合削防）→ 策略深度初現
🟣 史詩：高階操作 + 專屬 hex（削弱敵防 40%）→ 強力單點突破
🟡 傳說：精英池 + 專屬 final_stand（死亡逆轉）→ 戲劇性大招
🔴 神話：專屬 rebirth（復活）→ 打破死亡規則
```

## 3. 詳細機制說明

### 共享技能

#### 3.1 `atk_boost` — 攻擊強化（⚪🟢🔵）

```
效果：ATK = ATK × (1 + X%)
結算：Phase 1（技能修正階段）
範例（🔵稀有, ATK=37）：ATK = 37 × 1.21 = 45（卡面顯示「ATK +8」）
```

#### 3.2 `def_boost` — 防禦強化（⚪🟢🔵）

```
效果：DEF = DEF × (1 + X%)
結算：Phase 1（技能修正階段）
範例（🔵稀有, DEF=25）：DEF = 25 × 1.30 = 33（卡面顯示「DEF +8」）
```

#### 3.3 `shield` — 護盾（⚪🟢🔵🟣）

```
效果：受到的傷害 × (1 - X%)
結算：Phase 2（傷害計算時）
範例（🔵稀有, 對手打出傷害 20）：實際受到 20 × (1-0.26) = 15
```

#### 3.4 `counter` — 反擊（⚪🟢🔵）

```
效果：反彈受到傷害的 X% 給對手（額外傷害，不減少自己受到的傷害）
結算：Phase 4（後處理，HP 結算之後）
範例（🔵稀有, 受到傷害 20）：對手額外受到 20 × 38% = 8 傷害
```

#### 3.5 `madness` — 狂暴（🟢🔵🟣🟡）

```
效果：ATK = ATK × N，DEF = 0
結算：Phase 1（技能修正階段）
範例（🔵稀有, ATK=37, DEF=25）：ATK = 37 × 1.8 = 67, DEF = 0
高風險高回報：輸出暴增但完全無防禦
```

#### 3.6 `double_strike` — 二連擊（🔵🟣🟡）

```
效果：攻擊兩次，第一擊正常結算，第二擊 = 第一擊造成的傷害 × X%（額外傷害，不重新扣 DEF）
結算：Phase 2（傷害計算時）
範例（🔵稀有, ATK=37, 對手DEF=25）：
  第一擊傷害 = 37 - 25 = 12
  第二擊傷害 = 12 × 43% = 5
  總傷害 = 12 + 5 = 17
```

#### 3.7 `pierce` — 穿透（🔵🟣🟡）

```
效果：傷害 = ATK - DEF × (1 - X%)，削減對手 DEF 的有效值
結算：Phase 2（傷害計算時，替換傷害公式）
範例（🟣史詩, ATK=42, 對手DEF=34）：
  傷害 = 42 - 34 × (1-0.45) = 42 - 18.7 = 23
  （正常公式: 42-34=8）
對高 DEF 目標效果更好：對手 DEF 越高，被削減的絕對值越大
```

### 專屬技能

#### 3.8 `spite` — 怨恨（⚪ 專屬）

```
效果：死亡時，對對手造成已損失 HP × X% 的真實傷害
觸發：HP ≤ 0 時（Phase 3 死亡結算）
數值：X = 30%

範例（⚪凡品, maxHP=100, 死亡時已損失 65 HP）：
  反傷 = 65 × 30% = 20 真傷
  → 即使輸了也拉對手一把

特殊規則：
  - 真實傷害，無視對手 shield
  - 如果 final_stand 觸發（HP→1 而非死亡），spite 不觸發
  - 如果 rebirth 觸發（復活），spite 不觸發
  - 只在真正死亡時觸發一次
```

**設計意圖：** 白卡最可能輸，但 spite 確保「輸了也留痕跡」，降低抽到白卡的挫敗感。

#### 3.9 `momentum` — 動勢（🟢 專屬）

```
效果：若上回合受到傷害 > 0，本回合 ATK +X%
觸發：Phase 1（檢查上回合記錄）
數值：X = 25%

範例（🟢精良, ATK=26, 上回合受到 8 傷害）：
  ATK = 26 × 1.25 = 33（本回合 ATK +7）

特殊規則：
  - 第一回合無法觸發（沒有「上回合」）
  - 需要對戰引擎記錄「上回合是否受傷」
```

**設計意圖：** 「越打越強」的叙事感，讓玩家體會條件觸發的樂趣。

#### 3.10 `leech` — 蛭噬（🔵 專屬）

```
效果：造成傷害後，對手下回合 DEF -X%
觸發：Phase 3（傷害結算後，如果造成了傷害）
數值：X = 20%

範例（🔵稀有 造成 15 傷害, 對手 DEF=25）：
  → 對手下回合 DEF = 25 × 0.80 = 20
  → 下回合打對手更容易

特殊規則：
  - 如果自己本回合沒造成傷害（ATK ≤ DEF 且最低傷害 1 仍算造成傷害），不觸發
  - 效果持續到下回合結束
  - 不可疊加：每回合以對手原始 DEF 為基礎重新計算
```

**設計意圖：** MVP 中唯一的跨回合效果，讓玩家初步體驗「現在犧牲 → 未來收益」的策略感。

#### 3.11 `hex` — 詛咒（🟣 專屬）

```
效果：對手 DEF -X%
觸發：Phase 1（修改對手 DEF）
數值：X = 40%

範例（🟣史詩 vs 對手 DEF=34）：
  對手 DEF = 34 × 0.60 = 20（-14）
  → 自己正常攻擊的傷害大幅增加

特殊規則：
  - 只影響 DEF，不影響 ATK
  - 如果對手有 madness（DEF=0），hex 無額外效果
```

**設計意圖：** 純粹的攻擊輔助技能。與 pierce（穿透自己的傷害計算）不同，hex 真的降低對手 DEF 數值，影響所有傷害來源（包括 counter 反彈的計算基礎）。

#### 3.12 `final_stand` — 絕地反擊（🟡 專屬）

```
效果：當 HP 將降至 0 以下時，改為 HP = 1，並對對手造成 ATK × X% 真傷。每場只觸發一次。
觸發：Phase 3（HP 結算時）
數值：X = 80%

範例（🟡傳說, ATK=55, 當前HP=8, 受到傷害 20）：
  HP = 8 - 20 = -12 → 觸發 final_stand → HP = 1
  反擊傷害 = 55 × 80% = 44（真實傷害）
  → 瀕死一擊翻盤

特殊規則：
  - 每場對戰（5 回合）只能觸發一次
  - 觸發後 hasTriggeredFinalStand = true
  - 如果 HP 恰好降至 0（不是負數），仍觸發
  - 反擊傷害為真實傷害，無視 shield
```

**設計意圖：** MVP 中最具戲劇性的技能。「差點死了但活下來」比「一路碾壓」記憶更深刻（Peak-End Rule）。

#### 3.13 `rebirth` — 重生（🔴 專屬）

```
效果：首次 HP ≤ 0 時，HP 恢復為 maxHP × X%
觸發：Phase 3（HP 結算時，與 final_stand 同階段但不同稀有度不衝突）
數值：X = 50%（= 50 HP）

範例（🔴神話, HP=10, 受到 30 傷害）：
  HP = 10 - 30 = -20 → 觸發 rebirth → HP = 50
  → 相當於額外多了 50 點生命值

特殊規則：
  - 每場對戰只觸發一次
  - 復活後技能效果不重置（已經觸發過的一次性效果不會再觸發）
  - 復活不觸發 spite（spite 需要「真正死亡」）
  - 復活後 HP=50，下回合繼續正常戰鬥
```

**設計意圖：** MVP 中唯一的「破規則」技能 —— 打破「HP=0 就是死亡」的基本法則。作為 🔴 神話唯一技能，rebirth 最直觀、最震撼、實作最簡單。

## 4. 結算順序（MVP 簡化版）

```
Phase 1: 技能修正階段
  → momentum 檢查（上回合受傷 → ATK +25%）
  → leech debuff 生效（上回合被 leech 命中 → 本回合 DEF -20%，以原始 DEF 為基礎）
  → hex 結算（對手 DEF -40%）
  → atk_boost, def_boost 結算
  → madness 結算（ATK ×N, DEF=0）

Phase 2: 傷害計算
  → 正常: damage = ATK - DEF (min 1)
  → pierce: damage = ATK - DEF×(1-X%)
  → double_strike: 第一擊正常 + 第二擊 = 第一擊傷害×X%（不重新扣DEF）
  → shield: damage × (1 - X%)

Phase 3: HP 結算
  → 扣減 HP
  → 如果 HP ≤ 0 且持有 rebirth 且未觸發過 → HP = maxHP × 50%
  → 如果 HP ≤ 0 且持有 final_stand 且未觸發過 → HP = 1 + 反擊傷害
  → 如果 HP ≤ 0（真正死亡）且持有 spite → 對手受真傷

Phase 4: 後處理
  → counter 反彈
  → leech 設定（對手下回合 DEF -X%）

Phase 5: 回合結束判定
  → 某方 HP ≤ 0 → 戰鬥結束
  → 5 回合結束 → HP 高者勝（平手 = 雙方敗）
```

### MVP 不需要處理的邊界

完整版有 12 種複雜邊界互動（mirror vs mirror、eclipse vs eclipse、paradox vs paradox、domain vs domain 等）。MVP 的 13 種技能之間**沒有任何需要特殊處理的互動邊界**，所有技能效果線性結算，大幅降低對戰引擎複雜度。

唯一需要注意的互動：
- `final_stand` / `rebirth` 觸發時阻止 `spite`（不同稀有度不會同時出現，但邏輯要正確）
- `hex` + 對手 `madness`（DEF 已經是 0，hex 無效果）—— 自然處理，不需特殊邏輯

## 5. Firestore 技能欄位

```javascript
battle: {
  atk: 37,                          // 基礎 ATK（不含技能加成）
  def: 25,                          // 基礎 DEF（不含技能加成）
  skillMechanic: "double_strike",   // 13 種機制 ID 之一（受稀有度限制）
  skillParams: {                    // 該機制在此稀有度的參數
    secondHitRatio: 0.43            // 依機制不同，key 不同
  },
  skillName: "肉球連擊",            // AI 生成的獨特技能名稱
  skillDesc: "揮出兩記致命肉球，第二擊額外傷害 5",  // AI 生成的效果描述
}
```

> MVP 不需要 `secondSkillMechanic` 等 doppelganger 相關欄位。v2 加入時再擴展。

### 各機制的 skillParams 格式

#### 共享技能

| 機制 | skillParams |
|------|-------------|
| `atk_boost` | `{ atkBoostRatio: 0.21 }` |
| `def_boost` | `{ defBoostRatio: 0.30 }` |
| `shield` | `{ damageReductionRatio: 0.26 }` |
| `counter` | `{ counterRatio: 0.38 }` |
| `madness` | `{ atkMultiplier: 1.8 }` |
| `double_strike` | `{ secondHitRatio: 0.43 }` |
| `pierce` | `{ pierceRatio: 0.45 }` |

> 以上數值以 🔵 稀有為例，實際數值取決於該技能出現的稀有度。

#### 專屬技能

| 機制 | skillParams |
|------|-------------|
| `spite` | `{ damageRatio: 0.30 }` |
| `momentum` | `{ atkBoostRatio: 0.25 }` |
| `leech` | `{ defReductionRatio: 0.20 }` |
| `hex` | `{ defReductionRatio: 0.40 }` |
| `final_stand` | `{ counterRatio: 0.80, triggered: false }` |
| `rebirth` | `{ reviveRatio: 0.50, triggered: false }` |

> `triggered: false` 的技能（final_stand, rebirth）由對戰系統管理狀態。

## 6. AI 技能選擇指引

Claude API 在生成卡片時，**只從該稀有度可用的 MVP 技能池中選擇**最符合照片 + 咒語語境的技能。

### 選擇原則

```
1. 稀有度限制（強制）：
   - 系統根據 Section 1 的可用矩陣，過濾出該稀有度可用的技能列表
   - AI 只從過濾後的列表中選擇

2. 語義契合（優先）：
   - 拍火焰 → madness, pierce
   - 拍食物 → spite（「吃了也要留下味道」）, shield
   - 拍動物 → double_strike, momentum, spite
   - 拍建築 → shield, counter, def_boost
   - 拍電子產品 → leech, hex
   - 拍不死鳥/涅槃感 → rebirth（🔴限定）
   - 拍具有「最後一搏」感 → final_stand（🟡限定）

3. 咒語導向（覆蓋照片預設）：
   - 拍食物 + 咒語「爆炸吧」→ 可以選 madness 而非 shield
   - 拍建築 + 咒語「碎裂」→ 可以選 pierce 而非 shield

4. 專屬技能優先提示：
   - 當稀有度有專屬技能時，AI 應有一定傾向選擇專屬技能
   - 🔴 神話 = rebirth（唯一選擇，無需 AI 決策）
   - 其他稀有度：建議 40% 機率選專屬，60% 選共享（由 AI 自然判斷）

5. 多樣性：
   - 不要每次拍同類物體都選相同技能
   - 適度驚喜：偶爾選出乎意料但仍能自圓其說的組合
```

### 🔴 神話技能選擇（MVP 簡化）

MVP 中 🔴 神話只有 `rebirth` 一個技能，AI 不需要做選擇。系統直接指定 `skillMechanic = "rebirth"`，AI 只負責命名。

```
範例：
  拍日出 → rebirth 命名為「曙光再臨」
  拍嬰兒 → rebirth 命名為「初生之力」
  拍鳳凰 → rebirth 命名為「涅槃浴火」
```

## 7. v2 升級路線

### 技能回歸優先順序

| 優先度 | 技能 | 理由 |
|--------|------|------|
| P1 | dodge, lifesteal, drain, fortress | 共享池基礎擴充，對戰引擎改動小 |
| P1 | stubborn, taunt, trap, gambit | 每稀有度補齊第 2 個專屬 |
| P2 | mirror, siphon, eclipse, phoenix | 高階專屬，結算複雜度中等 |
| P3 | apocalypse, overkill, doppelganger | 需要特殊結算階段（Phase -1, Phase 5, 雙技能） |
| P4 | domain, paradox | 需要完全獨立的結算路徑（交換卡牌、逆轉勝利條件） |

### 資料結構前瞻

MVP 的 Firestore 結構已預留 v2 擴展空間：
- `battle` 物件可直接加入 `secondSkillMechanic` 等 doppelganger 欄位
- `skillParams` 的 key-value 格式天然支持新技能的參數
- `triggered: false` 模式可複用於 overkill 等一次性技能

**無需任何資料遷移即可升級到完整 30 技能版本。**
