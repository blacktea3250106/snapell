# VoxCards 開發路線圖

> **原則：** 一人團隊，分階段驗證。每個階段都有明確的核心假設和驗證指標。

---

## 階段總覽

```
MVP-v0（製卡驗證）          MVP-v1（對戰驗證）
  拍照+咒語→AI生成卡片        好友對戰+結算引擎
  收藏瀏覽                    13技能完整結算
  簡化卡面渲染                 場地卡系統
  ───────────────→          ───────────────→
  驗證：製卡好不好玩？          驗證：對戰好不好玩？
```

---

## MVP-v0：製卡驗證

### 核心假設

**「拍照 + 咒語 → AI 生成獨特卡片」這件事本身就夠好玩，玩家會反覆製卡並主動分享。**

### 驗證指標

| 指標 | 目標 | 怎麼測 |
|------|------|--------|
| 製卡留存 | 次日回來再製卡 ≥ 40% | Firebase Analytics |
| 分享率 | ≥ 10% 的卡片被截圖/分享 | 分享按鈕點擊率 |
| 平均製卡數 | 首週 ≥ 8 張 | Firestore 統計 |
| 咒語填寫率 | ≥ 70% 製卡有填咒語 | Firestore 統計 |

### 功能範圍

| 做 | 不做 |
|----|------|
| 拍照 + 文字咒語輸入 | 對戰系統全部 |
| Claude API（照片分析 + 咒語分析 + 卡片生成） | 場地卡全部 |
| 稀有度純機率抽選 + 新手保底 | 結算引擎全部 |
| ATK/DEF 生成（張力方向） | 好友房間 / 邀請碼 |
| 技能分配（AI 選 + 動態命名） | 即時同步 / 心跳 / 斷線 |
| 前端 CustomPainter 卡面渲染（不用 Sharp） | Sharp 後端卡面合成 |
| 煉成動畫（簡化版） | 對戰動畫全部 |
| 卡片列表 + 卡片詳情 | 牌庫構築 |
| Firebase Auth + Firestore + Storage | 複雜防作弊 |
| 基礎分享功能（截圖分享） | 戰績系統 |

### 精簡決策

| 原 MVP 設計 | v0 精簡方案 | 省了什麼 |
|-------------|-----------|---------|
| 6 級稀有度 | **保留 6 級** | 稀有度是翻卡驚喜的核心，不能砍 |
| 13 種技能（7共享+6專屬） | **7 種共享技能**，專屬技能延後到 v1 | 少 6 種技能的命名邏輯，結算反正不做 |
| Sharp 後端合成卡面圖 | **前端 CustomPainter 直接渲染** | 省掉 Sharp 依賴 + 後端合成 + Storage 卡面圖 |
| 6 種煉成特效 | **3 種**（無特效/光暈/全螢幕） | 白綠=無，藍紫=光暈，金紅=全螢幕 |
| 內容審核（mild/severe） | **保留**（上線必須有） | 不能省 |

### 文件清單

```
MVP-v0/
├── CARD_CREATION_v0.md    — 製卡系統規格（精簡版）
├── SKILL_SYSTEM_v0.md     — 技能系統（7種共享，僅定義不結算）
└── CARD_VISUAL_v0.md      — 卡面渲染規格（前端 CustomPainter）
```

## MVP-v1：對戰驗證

### 核心假設

**「用自己拍照生成的卡片和朋友對戰」有足夠的策略深度和社交樂趣，玩家會反覆對戰。**

### 前置條件

MVP-v0 數據達標（製卡留存 ≥ 40%，證明製卡本身夠好玩）

### 驗證指標

| 指標 | 目標 | 怎麼測 |
|------|------|--------|
| 對戰完成率 | 開始對戰→打完 ≥ 80% | 非斷線的完成率 |
| 對戰後製卡 | 打完一場後 30 分鐘內製卡 ≥ 30% | 事件序列分析 |
| 重複對戰 | 同一對好友一天內 ≥ 2 場 | Firestore 統計 |
| 場地卡滿意度 | 「場地卡讓對戰更有趣」同意 ≥ 70% | 問卷 |

### 功能範圍（在 v0 基礎上新增）

| 新增 | 仍然延後 |
|------|---------|
| 好友對戰（邀請碼建房） | 隨機配對 / 排位 |
| 牌庫選擇（從收藏選 10 張） | 觀戰 / 回放 |
| 完整結算引擎（Phase 1~4） | 對戰獎勵 |
| 13 種技能完整結算（回補 6 種專屬） | 分享戰報 |
| 8 張場地卡 | 段位 / 賽季 |
| Firestore 即時同步 | AI 對戰（可評估提前） |
| 基礎防作弊（Server 結算） | 精緻動畫（先用簡單 UI） |
| 超時 / 斷線處理 | 卡組預設管理 |
| 戰績更新 | 完整心跳機制（先簡單超時判負） |
| 4 種特殊事件（文字提示即可） | 特殊事件專屬動畫 |

### v0 → v1 升級改動

| 項目 | v0 | v1 改動 |
|------|-----|--------|
| 技能 | 7 共享（僅卡面文字） | +6 專屬 = 13 種完整結算 |
| 卡面 | 前端 CustomPainter | 不變（夠用） |
| Firestore | cards + users | +battles + moves + inviteCodes |
| Cloud Functions | createCard | +createBattleRoom, joinBattleRoom, submitDeck, submitMove |
| Security Rules | 基礎 | +move 鎖定 + 場地卡加密 + 卡片歸屬驗證 |

### 文件清單

```
MVP-v1/
├── BATTLE_SYSTEM_v1.md      — 對戰結算引擎
├── FIELD_CARDS_v1.md         — 8張場地卡完整定義
├── SKILL_SYSTEM_v1.md        — 13種技能完整結算
└── UPGRADE_FROM_v0.md        — v0→v1 升級指南（資料遷移、新增端點）
```

## 時間線（粗估）

```
        v0 製卡驗證              v1 對戰驗證
  ┌────────────────────┐  ┌────────────────────┐
  │ 後端 + AI 串接       │  │ 結算引擎            │
  │ 前端製卡流程          │  │ 場地卡系統           │
  │ 卡面渲染             │  │ 對戰 UI              │
  │ 收藏管理             │  │ 即時同步              │
  │ 封測（自己+少數朋友）  │  │ 封測（更多朋友）       │
  └────────────────────┘  └────────────────────┘
                ↓ 數據達標後才進入 v1
```
