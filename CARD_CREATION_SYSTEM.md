# VoxCards 製卡系統完整設計文件

## 萬物皆卡，以文鑄魂，以聲封印

> **你拍的每一張照片是卡片的身體。你寫的每一句咒語是卡片的靈魂。**
> **而你的聲音，會被所有與你交手的對手聽到。**

---

# 一、設計核心

## 一句話

```
製卡不是抽獎。是創作。是表演。是留給未來對手的一封信。
```

## 製卡的三層樂趣

| 層次 | 樂趣來源 | 對應的心理機制 |
|------|---------|--------------|
| 探索 | 看到現實中的東西想「這能變成什麼卡？」 | 好奇心（Information Gap） |
| 表演 | 想出一句很酷/很搞笑的咒語，知道對手會聽到 | 社交表現慾（Self-presentation） |
| 驚喜 | 煉成結果揭曉的瞬間 | 可變比率強化（Variable Ratio Reinforcement） |

## 為什麼咒語是核心

```
其他卡牌遊戲：抽卡 = 按按鈕 → 等結果
VoxCards：製卡 = 拍照 + 寫咒語 → 等結果

差別在哪？

按按鈕：每次都一樣。第 100 次跟第 1 次沒有區別。
寫咒語：每次都不同。因為你不會重複寫同一句話。

而且——你的咒語會被封印在卡片裡。
對戰翻卡時，對手會看到你的文字。
如果你還錄了音，對手會聽到你的聲音。

這意味著：
  你不是在「給系統輸入資料」
  你是在「為這張卡鑄入靈魂」
  你是在「留一段話給未來某個不認識的人」

→ 這就是為什麼第 100 次寫咒語依然有趣
→ 因為動機不是「系統獎勵」而是「別人會看到/聽到」
```

## 咒語的兩層設計：文字鑄魂，語音封印

```
文字 = 卡片的靈魂（影響數值）
  咒語文字決定稀有度加成、ATK/DEF 方向、技能扭轉
  所有玩家平等——打字就能拿滿分
  不需要語音辨識，零技術風險

語音 = 卡片的聲音（純社交價值）
  寫完咒語後，可選擇錄音
  螢幕顯示咒語文字當提詞 → 語音和文字天然一致
  語音不影響任何數值
  對戰翻卡時，有語音的卡播放聲音 → 更有衝擊力
  沒語音的卡顯示文字動畫 → 一樣有咒語，只是安靜

為什麼這樣設計：
  ① 公平——害羞的人、安靜環境、聽障玩家不受懲罰
  ② 簡單——不需要 STT（語音轉文字），零轉錄錯誤
  ③ 一致——語音有提詞，不會出現語音和文字對不上的情況
  ④ 誠實——語音的價值來自「對手聽到你的聲音」，不是系統多給 5 分
```

# 二、完整製卡流程

## 流程總覽

```
📷 拍攝（2 秒）
  ↓
✏️ 咒語文字（3~8 秒）— 打字輸入
  ↓
🎙️ 注入靈魂（可選，3~5 秒）— 錄音，螢幕提詞
  ↓
📤 上傳（1 秒）
  ↓
🌀 煉成動畫（4 秒）— 三階段揭示
  ↓
📊 分數構成（2 秒）— 照片力/咒語力/命運值
  ↓
🃏 卡片誕生 — 品鑑時刻
```

---

## 第一步：📷 拍攝

```
打開相機 → 對準任何東西 → 按下快門

拍攝目標：現實中的任何事物
  你家的貓、午餐的拉麵、路邊的消防栓、天上的雲、你的鞋子
  什麼都行。什麼都能變成卡。

技術：
  Flutter camera 套件
  存成 JPEG（壓縮到 500KB 以內）
  需要相機權限

拍照的遊戲意義：
  照片決定三件事——
    ① 卡面視覺主體（你的照片就是卡面）
    ② 技能系別（照片類別 → 技能池）
    ③ ATK/DEF 傾向的一部分（照片穩定度/構圖影響基礎分）
```

### 照片類別 → 技能池對應

```
AI（Claude Vision）分析照片內容，自動分類：

  🔥 火焰/光源/太陽/爆炸     → 火焰系技能池（攻擊增幅）
  💚 食物/飲料/廚房           → 治癒系技能池（回復續航）
  ⚡ 動物/寵物/交通工具       → 迅捷系技能池（先制連擊）
  🛡️ 建築/牆壁/石頭/地面      → 防禦系技能池（減傷反擊）
  ⚙️ 電子產品/機器/工具       → 干擾系技能池（削弱對手）
  🌿 自然風景/植物/天空       → 自然系技能池（環境互動）
  🎭 人/人群/自拍             → 混沌系技能池（高風險高回報）
  ❓ 無法分類/抽象            → 全技能池隨機

注意：
  這個對應關係不會直接告訴玩家
  玩家透過多次製卡自己歸納出規律
  歸納本身就是樂趣（IKEA 效應）
```

### 照片也影響 ATK/DEF 傾向

```
照片類別除了決定技能池，也「輕微」影響 ATK/DEF 基礎傾向：

  🔥 火焰系物體 → ATK 微偏高（+5%）
  🛡️ 建築類物體 → DEF 微偏高（+5%）
  ⚡ 動物類 → ATK 微偏高（+3%）
  💚 食物類 → DEF 微偏高（+3%）
  其他 → 無偏移

  這個偏移很小（5% 以內）
  真正決定 ATK/DEF 分配的是咒語（見下一步）
  照片只是「底色」，咒語是「筆觸」
```

## 第二步：🗣️ 咒語 — 注入靈魂

### 咒語的雙重意義

```
功能層面（文字）：
  咒語文字影響卡片的稀有度（咒語分數 0~30）
  咒語文字影響 ATK/DEF 分配比例（情緒張力方向）
  咒語文字影響技能「扭轉」（語義衝突時可能改變技能系別）

靈魂層面（語音，可選）：
  語音會被永久封印在卡片中
  對戰翻卡時，對手會聽到你的聲音
  你的聲音 = 這張卡的身份標記
  語音不影響任何數值 — 純社交價值
```

### 兩步流程：先寫字，再錄音（可選）

```
Step 1：輸入咒語文字（必要）
  打字輸入你的咒語
  這段文字決定所有評分
  上限 30 分，跟是否錄音無關

Step 2：注入靈魂（可選）
  寫完咒語後，畫面提示：
  「要為這張卡注入靈魂嗎？」
  [🎙️ 錄音] [跳過]

  選擇錄音 → 螢幕顯示咒語文字當提詞（像 KTV 字幕）
  → 玩家自然會照著念（也可以加油添醋）
  → 語音和文字天然一致
  → 不需要語音轉文字（STT）

  選擇跳過 → 這張卡只有文字咒語
  → 對戰時顯示文字動畫，不播放聲音
```

### 三種模式對照

| 模式 | 操作 | 咒語分上限 | 對戰表現 | 定位 |
|------|-----|-----------|---------|------|
| ⌨️ 文字模式 | 打字 | 30 | 顯示文字動畫 | 基礎模式，人人平等 |
| 🗣️ 語音+文字 | 打字 + 錄音 | 30（一樣） | 播放語音 + 顯示文字 | 社交表演模式 |
| 🤖 自動模式 | 跳過 | 固定 11 | 顯示「（無咒語）」 | 懶人快速模式 |

```
關鍵設計：
  語音和文字的數值上限完全相同（30）
  語音的唯一差異是對戰時「對手聽到聲音」
  → 不是系統獎勵語音，而是聲音本身就比文字更有衝擊力
  → 社交價值驅動，不是數值驅動

自動模式觸發：
  打開咒語輸入 → 5 秒沒打字 → 自動模式（固定 11）
  → 不認真就低分，公平誠實
```

### 咒語視覺演出：粒子反應

```
寫咒語時，你拍的照片不是靜止的——它會「反應」：

打字階段：
  每打一個字 → 文字化為能量粒子飄向照片
  照片周圍的粒子隨字數增加越來越密集
  打完確認 → 光點聚合吸入照片

錄音階段（如果選擇錄音）：
  你開始喊 → 照片周圍粒子被聲音激活
  你的聲音越大 → 粒子飄動越劇烈
  喊完瞬間 → 所有粒子被吸入照片

  視覺上：像是你在對照片「施法」
  照片裡的物體彷彿被你的聲音喚醒

無論哪種：
  粒子吸入照片 → 照片縮成卡片大小 → 進入煉成階段

→ 不改變任何遊戲數據
→ 純視覺演出
→ 但把「輸入咒語」從「填表」變成「對卡片施法」
→ 儀式感
```

### 咒語評分（0~30）— 純文字評分

```
Claude API 分析咒語文字，兩個維度各 0~15：

🎯 契合度（0~15）：咒語跟照片有多相關
  「燃燒吧小貓」（拍的貓）→ 12（提到貓的特徵 + 火焰意象）
  「今天天氣真好」（拍的貓）→ 3（跟貓無關）
  「肉球之下無人倖存」（拍的貓）→ 14（精準描述 + 有張力）

🎪 創意（0~15）：說法獨特程度 + 情緒張力
  「毀滅吧」→ 5（太常見）
  「肉球碾壓一切」→ 12（有畫面感，具體）
  「咕咕咕咕咕咕」→ 10（拍鴿子時很有趣，但拍貓就低分）

  重複懲罰：
    AI 記錄最近 50 次咒語，重複率越高創意分越低
    第一次「毀滅吧」→ 5
    第三次「毀滅吧」→ 1

注意：
  評分完全基於文字，跟是否錄音無關
  語音模式和文字模式的上限完全相同（30）
  → 公平。害羞的人、安靜環境、聽障玩家不受任何數值懲罰
```

### 咒語影響 ATK/DEF 分配

```
AI 分析咒語的「情緒張力方向」，決定 ATK/DEF 比例：

  張力外放（攻擊性、破壞性、狂暴感）→ ATK 佔比高
    例：「毀滅一切！」「粉碎他！」「燃燒吧！」

  張力內斂（守護性、防禦性、沉穩感）→ DEF 佔比高
    例：「守護我吧」「堅不可摧」「安息」

  張力中性（平衡、哲學、描述性）→ 均勻分配
    例：「萬象歸一」「生命的循環」「就是這樣吧」

  不是偵測關鍵字，而是整句話的語氣和意象
  → 鼓勵自然表達，不是機械輸入
```

### 咒語的「扭轉」效果

```
當咒語語義跟照片類別「衝突」時，技能池會發生偏移：

基本規則：
  照片和咒語語義一致 → 技能方向穩定（主分類 85%）
  照片和咒語語義衝突 → 技能方向混合（主分類 40% / 咒語方向 40% / 隨機 20%）

範例：
  拍貓（動物→迅捷系）+ 喊「疾風突刺」（迅捷語義）
    → 迅捷系 85%，隨機 15%
    → 高度可預測

  拍貓（動物→迅捷系）+ 喊「烈焰焚天」（火焰語義）
    → 迅捷系 40%，火焰系 40%，隨機 20%
    → 可能出現意外組合

→ 高契合 = 穩定產出（你知道你要什麼）
→ 低契合 = 冒險突變（可能驚喜，可能意外）
→ 「契合度」從純分數項變成玩家主動選擇的風險維度
```

### 咒語內容審核

```
咒語文字上傳時經過 AI 內容審核（Cloud Function）：

  咒語文字 → 直接審核（不需要 STT，因為文字是玩家手動輸入的）

審核項目：
  髒話 / 人身攻擊 / 色情 / 政治敏感

文字違規處理：
  輕度（髒話）→ 咒語文字自動替換為「***」
  重度（攻擊/色情）→ 咒語文字替換為「（咒語已封印）」

語音違規處理：
  因為語音有提詞（顯示咒語文字），大部分語音會跟文字一致
  但玩家可能在錄音時臨時加料（文字正常但語音加髒話）
  → 語音審核方案：
    a. 信任為主 — 不審核語音，靠檢舉機制（低成本，推薦 MVP 階段）
    b. 事後抽查 — 被檢舉後再審核語音內容
  → 被檢舉成立 → 原聲替換為系統默認音效「（咒語已封印）」

  不影響卡片數值，只影響對戰播放
  → 玩家不會因為審核而「浪費一次製卡」
```

---

## 第三步：📤 上傳

```
寫完咒語（+ 可選錄音）→ 自動上傳

上傳內容：
  照片 → Firebase Storage: photos/{userId}/{timestamp}.jpg
  語音 → Firebase Storage: voices/{userId}/{timestamp}.m4a（有錄音才有）
  咒語文字 → 直接傳入 Cloud Function

觸發 Cloud Function 開始後續處理：
  ④ 照片分析（Claude Vision API）
  ⑤ 咒語文字評分（Claude API — 契合度 + 創意）
  ④ 和 ⑤ 同時進行

  注意：不需要語音轉文字（STT）
  語音只是存儲到 Storage，等對戰時播放
  所有評分都基於玩家手動輸入的文字

耗時：上傳本身 < 1 秒（照片 500KB + 音檔 ~100KB）
```

## 第四步：🌀 煉成 — 三階段揭示動畫

**這是全遊戲最精心設計的 4 秒鐘。**

### 為什麼煉成動畫重要

```
成癮的多巴胺高峰不在「獲得獎勵」的瞬間
而在「即將獲得獎勵」的等待中

煉成動畫 = 把「等 AI 跑完」的無聊等待
           變成一場 4 秒鐘的情緒過山車
```

### 第一階段：卡框顯現（1.5 秒）

```
視覺：
  螢幕中央，一張背面朝上的卡片在旋轉
  卡片邊框的顏色慢慢浮現
  從模糊到清晰

框色含義（玩家需要自己摸索）：
  灰色卡框 → 85% 凡品/精良，15% 稀有
  彩色卡框 → 70% 稀有/史詩，25% 傳說，5% 精良
  金色卡框 → 80% 史詩/傳說，20% 神話

關鍵設計：
  灰框偶爾出稀有 → 「灰框也有希望！」
  彩框偶爾出精良 → 「彩框不是保證...」→ 下次看到彩框更緊張
  → 每一次都有懸念

玩家情緒：
  看到灰框 → 「嗯... 可能普通... 但上次灰框出了稀有！」
  看到彩框 → 「！！！是彩的！但上次彩框出了精良...」
  看到金框 → 「金的金的金的！！！」
```

### 第二階段：技能系浮現（1.5 秒）

```
視覺：
  卡片正中央浮出技能系的圖示
  🔥 / 💚 / ⚡ / 🛡️ / ⚙️ / 🌿 / 🎭

  圖示帶有對應的色彩光暈
  火焰系 = 橙紅光暈，治癒系 = 綠色光暈...

玩家情緒：
  「火焰系！我缺這個！」→ 興奮
  「治癒系... 不是我要的... 但也行」→ 微失落但接受
  「混沌系？！不知道會出什麼技能！」→ 好奇
```

### 第三階段：翻卡（1 秒）

```
視覺：
  卡片翻轉 → 正面朝上
  所有資訊同時揭曉：

    卡片名稱
    卡面圖像（你的照片 + 處理效果）
    ⚔️ ATK  🛡️ DEF
    ✨ 技能名稱
    稀有度標記

  稀有度特效（按等級遞增）：
    ⚪ 凡品 → 無特效，安靜
    🟢 精良 → 輕微綠色光暈
    🔵 稀有 → 藍色粒子爆發
    🟣 史詩 → 紫色光柱 + 螢幕震動
    🟡 傳說 → 金色全螢幕爆裂 + 音效
    🔴 神話 → 螢幕碎裂特效 + 暗紅光芒 + 專屬音效 + 震動

  → 特效本身就是稀有度的「宣告」
  → 不需要看文字就知道出了什麼等級
```

## 第五步：🃏 卡片誕生 — 品鑑時刻

### 煉成回饋：顯示分數構成

```
翻卡後，卡片下方簡短顯示分數構成（2 秒後自動淡出）：

  ┌──────────────────────┐
  │  ✨ 煉成完成           │
  │                      │
  │  📷 照片力：████████░ 24 │
  │  💬 咒語力：██████░░░ 18 │
  │  🎲 命運值：██████░░░ 19 │
  │                      │
  │  → 總分 61 → 🔵 稀有    │
  └──────────────────────┘

為什麼要顯示：
  玩家看到「咒語力：6」→ 知道是因為自己打了「衝啊」
  不會怪系統，會怪自己 →「下次認真點」或「算了白卡就白卡」
  兩種反應都是健康的 — 因為他理解了因果關係

  玩家看到「咒語力：24」→ 知道這句咒語寫得好
  成就感和掌控感 →「我知道怎麼出好卡了」

設計細節：
  只顯示三大項的分數，不顯示細項（契合度/創意）
  → 避免「解題化」— 玩家不需要知道哪個子維度得幾分
  → 只需要知道「咒語整體有多強」
  數字 + 進度條，直覺好懂
  2 秒後淡出 → 不打斷品鑑體驗
```

### 卡片放大展示

# 四、稀有度系統

## 機率分布

| 稀有度 | 代號 | 基礎機率 | 最大加成上限 | 情緒設計 |
|--------|------|---------|------------|---------|
| 🔴 神話 | Mythic | 0.5% | 2.5% | 「（尖叫）（手抖）（截圖）」 |
| 🟡 傳說 | Legendary | 3.0% | 7.0% | 「不可能吧！！！」 |
| 🟣 史詩 | Epic | 10.0% | 16.0% | 「我的天！！」 |
| 🔵 稀有 | Rare | 17.0% | 25.0% | 「喔喔喔！」 |
| 🟢 精良 | Uncommon | 38.0% | 41.0% | 「不錯喔」 |
| ⚪ 凡品 | Common | 31.5% | 無加成 | 「哦，一張卡」 |

### 為什麼凡品少於精良

```
傳統遊戲：最低稀有度 = 最常見 → 「又出白了」= 負面基線
VoxCards：精良（38%）> 凡品（31.5%）→ 「出綠」= 正常體驗

  出綠（38%）→ 情緒中性，這是基線
  出白（31.5%）→ 「比平常差一點」→ 輕微失落但不崩潰
  出藍（17%）→ 「比平常好！」→ 正面情緒

  → 基線情緒被往上推了一格
  → 玩家的「正常體驗」是有色卡，不是白卡
  → 整體情緒曲線更健康
```

## 稀有度 → 戰鬥屬性

```
稀有度直接影響三件事：

① 屬性池（ATK + DEF 總和）
② 屬性池內的分配自由度
③ 技能強度版本

  稀有度    ATK+DEF 總和    分配彈性      技能版本
  ⚪ 凡品    30~45         45:55~55:45    弱版
  🟢 精良    40~58         40:60~60:40    弱版
  🔵 稀有    52~72         35:65~65:35    中版
  🟣 史詩    66~84         30:70~70:30    中版
  🟡 傳說    78~96         25:75~75:25    強版
  🔴 神話    90~100        20:80~80:20    強版

說明：
  分配彈性 = 咒語能把 ATK/DEF 推到多極端
  凡品卡被迫均衡（最多 55:45）
  神話卡可以極端（最多 80:20）→ 純攻或純防的卡只有高稀有度才有
  → 高稀有度不只是「數字大」而是「能做到低稀有度做不到的極端構築」
```

### 技能強度分三版

```
弱版（凡品/精良）：
  烈焰衝擊 → ATK +5（不是 +10）
  治癒之光 → 回復 8 HP（不是 12）
  堅壁 → DEF +10（不是 +15）

中版（稀有/史詩）：
  烈焰衝擊 → ATK +10
  治癒之光 → 回復 12 HP
  堅壁 → DEF +15

強版（傳說/神話）：
  烈焰衝擊 → ATK +15
  治癒之光 → 回復 18 HP
  堅壁 → DEF +20

→ 同一個技能，稀有度越高效果越強
→ 但不是「完全不同的技能」→ 低稀有度玩家不會感覺「我少了一個機制」
→ 只是數字差距
```

## 五重保底（Pity System）

```
第一重：漸進保底（Soft Pity）
  連續 N 次沒出 🟣 史詩以上 → 機率逐漸提升
  第 20 次 +15%，第 25 次 +40%，第 30 次 100%

第二重：新手保護
  第 1 次召喚：100% 🔵稀有以上
  第 2~5 次：機率逐漸遞減的加成

第三重：每日首抽加成
  每天第一次製卡：🟣 史詩以上 ×2，🟡 傳說以上 ×1.5

第四重：回歸獎勵
  超過 48 小時沒登入 → 下次製卡所有稀有度機率 +50%

第五重：情緒補償
  連續對戰 3 敗 → 下次製卡史詩以上機率 ×1.5

→ 玩家不知道保底存在
→ 觸發時感覺像「運氣終於來了」
→ 保底偽裝成命運
```

# 五、對戰中的咒語播放

## 這是製卡體驗的「完成」時刻

```
你製卡時喊的咒語，不是喊完就消失了。
它被封在卡片裡，等待被「釋放」的那一天。

那一天就是——你用這張卡對戰的時候。
```

## 播放時機與方式

```
對戰中翻卡的瞬間：

  倒數 3... 2... 1...
  雙方卡片同時翻開

  翻開的同時，三種情況：
    有語音的卡 → 播放咒語錄音（前 3 秒精華）+ 卡面顯示咒語文字
    只有文字的卡 → 卡面顯示咒語文字（浮現動畫）
    自動模式的卡 → 不顯示

  雙方咒語同時呈現：
    你看到/聽到對手的咒語
    對手看到/聽到你的咒語
    畫面：左半邊你的卡，右半邊對手的卡

  → 這一瞬間就是 VoxCards 跟所有其他卡牌遊戲不同的時刻
  → 你不只是在「看數字比大小」
  → 你在看/聽一個真實的人類對著一張照片寫了什麼鬼東西
```

## 咒語精華截取

```
原始咒語可能 3~10 秒
對戰中只播放前 3 秒（精華部分）

為什麼：
  對戰節奏快（每回合 ~11 秒）
  播放 10 秒咒語會拖慢節奏
  大部分咒語的精華在前 3 秒（「毀滅吧——！」後面通常是喘氣聲）

完整咒語可以在：
  卡片詳情頁 → 完整播放
  對戰回顧 → 完整播放
  個人展櫃 → 完整播放
```

## 咒語反應系統

```
對戰結束後，雙方可以對對手的咒語標記反應：

可選反應（每場只能標記一個）：
  😂 笑死
  🔥 好猛
  😱 嚇到
  👏 佩服

標記後：
  卡片主人收到通知：「你的『垃圾桶之王降臨！』獲得了 😂 反應」
  卡片詳情頁累積顯示：「😂 ×15  🔥 ×3  👏 ×1」

→ 你知道別人聽到你的咒語有什麼反應
→ 這是「表演→觀眾回饋」的完成迴路
→ 收到 😂 多的咒語 → 你知道什麼樣的咒語受歡迎
→ 下次製卡時更有動力想一句更厲害的
```

## 語音 vs 文字的體驗差異

```
語音卡：翻卡時播放真人語音 + 顯示文字 → 極強的臨場感和個人特色
文字卡：翻卡時浮現文字動畫 → 有內容，但沒有「聲音的衝擊力」
自動卡：什麼都沒有 → 安靜

關鍵設計原則：語音不給數值獎勵

  語音卡和文字卡的數值完全相同（上限都是 30）
  語音的優勢純粹來自「聽到聲音比看到文字更震撼」
  這是物理現實，不是系統偏袒

  → 語音是表演型玩家的選擇
  → 文字是安靜型玩家的選擇
  → 兩者在數值上完全平等
  → 語音的社交價值（迷因感、咒語反應）會自然驅動部分玩家選擇錄音
  → 但不錄音不會變弱

為什麼這樣比「語音多 5 分上限」更好：
  ① 聽障玩家、語障玩家不受排斥
  ② 在圖書館、辦公室、深夜也能拿滿分
  ③ 不需要 STT（語音轉文字）→ 零技術風險、省 API 費用
  ④ 語音和文字天然一致（錄音時有提詞）→ 不會語音說 A 文字寫 B
```

# 七、製卡循環驅動力

## 短期驅動（每天）

```
每日首抽加成 → 「每天第一張特別準」→ 每天至少做一張
「再造一張」按鈕 → 剛完成的情緒直接導向下一次
今日最佳展示 → 主頁顯示「今日煉成最佳卡」→ 想超越它
```

## 中期驅動（每週）

```
卡組缺口 → 戰鬥結算提示「你缺防禦卡」→ 去拍建築物
賽季環境 → 場地卡池輪換 → 需要不同類型的卡組 → 製新卡
成就進度 → 「距離火焰收藏家還差 3 張」→ 去拍跟火有關的東西
```

## 長期驅動（每月）

```
賽季更替 → 新場地卡池 → 上賽季的卡組不一定適用 → 需要新卡
保底累積 → 「好像快要出傳說了...」→ 再做幾張
社交競爭 → 朋友的展櫃裡有神話卡 → 我也要
咒語表演 → 想出一句更酷的咒語 → 超越自己上一句
```

## 卡片分解系統

```
多餘的卡片可以分解：

  ⚪ 凡品 → 5 煉金碎片
  🟢 精良 → 15 煉金碎片
  🔵 稀有 → 40 煉金碎片
  🟣 史詩 → 100 煉金碎片
  🟡 傳說 → 300 煉金碎片
  🔴 神話 → 不可分解（太珍貴了）

碎片用途：
  100 碎片 → 下次製卡指定技能系別（不用看照片類別）
  300 碎片 → 下次製卡指定具體技能
  50 碎片 → 下次製卡命運值 +5

→ 凡品卡不是垃圾，是碎片來源
→ 做越多卡 → 碎片越多 → 可以「精準鍛造」
→ 每一張卡都有價值
```

# 十、設計關鍵指標

```
📏 單次製卡時長：10~22 秒（拍照 2s + 打字 3~8s + 可選錄音 3~5s + 上傳 1s + 煉成 4s + 品鑑 3~5s）
📏 情緒節點數：5~6 個（拍照選擇→寫咒語→可選錄音→框色暗示→技能系揭示→翻卡）
📏 內在動機來源：創作慾（咒語文字會被對手看到）+ 表演慾（語音會被對手聽到）
📏 外在動機來源：卡組缺口、稀有度追求、成就收藏、分解資源
📏 咒語對戰呈現：有語音播放前 3 秒 + 文字，無語音顯示文字動畫
📏 期待感持續時間：從確認咒語到翻卡 = 5~6 秒的純期待
📏 重玩動力：每次咒語不同 + 稀有度隨機 + 對手反應回饋
📏 長期留存設計：賽季輪換 + 分解累積 + 成就進度 + 社交展示
📏 公平性：語音/文字模式數值上限完全相同（30），零模式歧視
📏 技術簡化：不需要 STT（語音轉文字），省掉 Deepgram API 費用和轉錄錯誤風險
📏 因果透明：煉成結果顯示分數構成 → 玩家理解「投入→產出」的關係
📏 一人團隊可行性：核心功能零額外 API（語音純存儲），文字審核用已有的 Claude API
```
