# VoxCards — Figma UI 設計說明書

---

## 一、遊戲一句話介紹

> **對著你的貓大喊一句中二咒語，牠就變成一張能打架的卡牌。**

VoxCards 是一款「現實召喚」卡牌對戰手機遊戲。玩家用手機**拍攝現實中的任何東西**，再**對著它喊出一句咒語**，AI 就會把畫面和聲音融合，煉成一張獨一無二的卡牌。然後用這些卡牌跟全世界的玩家對戰。

---

## 二、核心循環（所有頁面都圍繞這個）

```
📷 拍攝現實 → 🗣️ 喊出咒語 → 🌀 AI 煉成 → 🃏 卡片誕生 → ⚔️ 對戰 → 📷 繼續召喚...
```

---

## 三、需要設計的核心頁面

### 🏠 3-1. 首頁 / 大廳

**功能**：玩家的主基地，所有入口
**需要的元素**：
- 玩家頭像 + 暱稱 + 段位（例：⚡ 高階咒術師）
- 代表卡片展示（玩家設定的最強卡）
- 今日召喚次數（例：3/5 剩餘）
- 快捷入口按鈕：
  - 「召喚」（拍照 + 念咒）— **最大最醒目的按鈕**
  - 「對戰」
  - 「卡組」
  - 「任務」
- 今日運勢指示（模糊的，例：🔥🔥🔥🔥🔥「火焰高漲的一天！」）
- 全服廣播條（跑馬燈）：「xxx 剛剛召喚了一張 🔴 神話卡！」

**設計方向**：
- 深色背景 + 魔法/神秘氛圍
- 召喚按鈕要有「按下去會有事情發生」的魔法感
- 不要太複雜，核心是兩個動作：召喚 & 對戰

---

### 📷 3-2. 召喚流程（4 步驟）

#### Step 1：拍照頁
- 全螢幕相機預覽
- 底部拍照按鈕（大圓形）
- 頂部提示文字：「拍下你想召喚的東西」
- 可選：相簿入口（左下角）
- 拍完後預覽 → 確認 or 重拍

#### Step 2：咒語頁
- 拍到的照片顯示在上方（縮小）
- 中央大區域：即時聲波動畫（正在錄音的視覺回饋）
- 倒數計時：3~10 秒
- 提示文字：「對著它喊出你的咒語！」
- 模式自動切換：
  - 開口說話 → 顯示聲波動畫 + 錄音中
  - 開始打字 → 彈出文字輸入框
  - 什麼都不做 → 5 秒後顯示「自動模式啟動」

#### Step 3：AI 煉成中（等待畫面，3~6 秒）
- 照片 + 咒語文字在中央
- 魔法陣旋轉動畫 / 煉金術動畫
- 進度文字：「正在融合...」「正在注入靈魂...」「即將誕生...」
- **這個頁面是製造期待感的關鍵**，要有儀式感

#### Step 4：卡片誕生（翻卡動畫）
- 卡片從背面翻到正面
- 翻卡動畫依品質不同而不同（見卡片設計章節）
- 翻開後展示完整卡片
- 底部按鈕：「查看詳情」「再召喚一次」「分享」

---

### 🃏 3-3. 卡組 / 收藏頁

**功能**：管理所有卡片
**需要的元素**：
- 網格瀏覽（3~4 列的卡片縮略圖）
- 篩選器：按品質 / 按等級 / 按時間
- 卡片縮略圖 75×100dp：照片 + 等級 + 邊框顏色表示品質
- 統一白色底卡牆（收藏冊感覺）
  - 高品質的卡在白色牆中會「自己發光」
  - 神話（唯一黑卡）在白色海洋中超醒目
- 點擊 → 進入卡片詳情頁

**設計重點**：
- 排列整齊 → 像翻開一本收藏冊
- 品質辨識靠邊框顏色 + 光效，不靠大小差異
- （例外：傳說 & 神話的縮略圖比其他大 8%）

### 🃏 3-4. 卡片詳情頁

- 全螢幕展示卡片（300×400dp）
- 卡片可互動：陀螺儀傾斜（史詩以上）
- 底部資訊欄：
  - 召喚者 / 召喚地點 / 召喚日期
  - ▶️ 播放原始咒語 按鈕
  - 卡片故事 / 描述
- 操作按鈕：設為代表卡 / 餵養 / 融合 / 分享

### ⚔️ 3-6. 對戰流程（BO3 標準流程）

#### 配對畫面
- 「尋找對手中...」
- 自己的代表卡 vs 問號剪影
- 配對成功 → 對手資訊快速展示（暱稱 + 段位 + 代表卡）

#### 選卡階段（15 秒）
- 系統隨機給 5 張卡（橫排或扇形排列）
- 玩家選 3 張 + 拖曳排列出場順序
- 倒數計時 15 秒
- 底部確認按鈕

#### 戰鬥回合（每回合 ~20 秒）

**❶ 翻卡（2 秒）**
- 畫面分左右：你的卡 vs 對手的卡
- 同時翻開 → 顯示卡名、等級、被動技能

**❷ 讀秒（3 秒）**
- 雙方卡片對峙
- 倒數 3... 2... 1...
- 被動技能效果浮現（如果有觸發的話）

**❸ 咒語時間（5 秒）**
- 畫面分割左右
- 你的這邊：即時聲波動畫（你在喊）
- 對手那邊：等待中動畫 / 蒙版
- 分數輸入方式同召喚流程的咒語頁

**❹ AI 評分結果（3 秒）**
- 三個維度分數逐一跳出：
  - 🎯 契合度：8/10 「提到了貓的特徵」
  - 🔥 氣勢：7/10 「聲音有力度」
  - 🎪 創意：6/10 「意象不錯」
- 雙方同時顯示

**❺ 結算（2 秒）**
- 戰鬥力數字碰撞動畫
- 勝方卡片把敗方彈飛
- 勝方亮燈 ✅

#### 對戰結算頁（最終比分）
- 比分：2:1 或 2:0
- 特殊事件標記（如：🔥 以下犯上 / ⚡ 險勝）
- 「本場最佳咒語」回放按鈕
- 按鈕：再來一局 / 分享戰報 / 返回

### 🔄 3-9. 餵養 & 融合頁

#### 餵養
- 顯示你的卡片 + 今天它想吃什麼（系統提示）
- 拍照按鈕：拍食物餵給卡片
- 結果動畫：卡片吃了 → +X 等級 / 卡片不喜歡 → +0

#### 融合
- 兩張卡拖入融合槽
- 中間顯示咒語錄製按鈕（念新咒語決定融合方向）
- 融合動畫 → 結果（成功 / 失敗）
- 失敗 → 殘骸碎片動畫

## 四、卡片視覺設計（核心中的核心）

### 設計原則：「統一風格 + 光的遞進」

```
所有卡片共用同一個白色底骨架（唯一例外：神話 = 黑底）
差異只在：邊框顏色 + 邊框光效 + 全息覆蓋面積
→ 收藏冊打開 = 整齊的白色卡牆
→ 高品質卡在白色牆中「自己會發光」
→ 神話是唯一的黑卡 → 在白色海洋中 = 視覺核彈
```

### 卡片統一尺寸

```
比例：3:4（直向）
邏輯像素：300 × 400 dp
縮略圖：75 × 100 dp
對戰用：150 × 200 dp
分享圖：1080 × 1350 px（Instagram 最佳比例）
```

### 六種稀有度視覺規格

#### ⚪ 凡品 — 霧面白卡
- **靈感**：Apple 包裝的極簡白
- 底色：純白 #FFFFFF
- 邊框：淡灰 #E5E7EB，1dp，圓角 12dp
- 卡名字體：Regular，深灰 #374151
- 分隔線：1dp 實線 #E5E7EB
- **無任何光效、無動畫**
- 光效面積：0%
- 整體感覺：乾淨、不丟人的白色收藏卡

#### 🟢 精良 — 微光白卡
- **靈感**：Apple Card 若有似無的光澤
- 底色：#FAFAFA
- 邊框：翠綠 #22C55E，1.5dp
- 邊緣光暈：極微弱綠色（opacity 10%）
- 卡名字體：Medium，#1F2937
- 角標：綠色小圓點 + 字母「U」
- 光效面積：0%（只有邊框顏色差異）
- 整體感覺：跟凡品 95% 一樣，但邊框是綠色

#### 🔵 稀有 — 銀色金屬卡
- **靈感**：Apple Card 鈦合金 / 高級信用卡
- 底色：銀灰微漸層 #F8FAFC → #EFF2F5
- 邊框：金屬反光漸層 #94A3B8 → #CBD5E1 → #94A3B8，2dp
- 雙層結構：外框 + 內線（Apple Card 感）
- 卡名浮雕效果：V 槽光影
- ATK/DEF/SPD 用工業標籤字體（全大寫）
- 角標：銀色菱形旋轉 45°
- 照片濾鏡：冷色調 + 銳化
- 光效面積：~5%
- 整體感覺：「第一張讓你覺得在拿真正收藏卡的」

#### 🟣 史詩 — 全息閃卡 ⚠️ 分享爆發點
- **靈感**：Pokemon TCG 全息閃卡
- 底色：白 #FAFAFA
- 邊框：**彩虹流光**（SweepGradient，4 秒轉一圈）
  - 色序：紅 → 黃 → 藍 → 粉 → 藍 → 紫 → 回紅
- 照片區全息覆蓋：彩虹光在照片上流動（opacity 35%）
- 陀螺儀互動：傾斜手機 → 彩虹光移動 + 卡片微傾斜 ±3°
- 卡名：ShaderMask 彩虹漸層文字
- 角標：彩虹全息圓形
- 光效面積：~40%（照片區全息 + 邊框流光）
- 整體感覺：「寶可夢閃卡！轉手機會反光！」
- **第一個讓人想錄影分享的等級**

#### 🟡 傳說 — 黃金聖光卡
- **靈感**：24K 金卡 / 奧斯卡小金人
- 底色：暖白漸層 #FFFCF5 → #FEF3C7
- 邊框：**金色流光**（三層結構，2 秒轉一圈）+ 四角旋轉光點
- 照片區全息：金色光澤（opacity 60%）
- 資訊區底色：微淡金（連資訊區都被金光浸染）
- **粒子效果**：20~25 個金色光點從底部上升
- 外發光：金色光暈 opacity 35%
- 卡名：金色漸層 + 金色外發光
- 角標：金色盾形
- 光效面積：~70%
- 整體感覺：「整張卡都在發光！有金色粒子在飄！」

#### 🔴 神話 — 暗黑禁忌之物 ⚠️ 全遊戲唯一暗色卡
- **靈感**：SCP 禁忌物品 / Panini Black 1/1
- 底色：**極深黑 #0A0A0F**（全遊戲唯一暗色卡！）
- 邊框：黑色 + **6~8 條紅色裂縫**（每張卡裂縫圖案不同！）
  - 裂縫外邊：暗紅 #DC2626
  - 裂縫內部：亮紅 #FF4444（岩漿感）
  - 裂縫脈動：opacity 在 0.5 和 1.0 之間跳動
- 照片區：壓暗 + 極高對比 + 暗紅色調 + 強暗角
- 整卡呼吸：微 scale 1.0 ↔ 1.003
- **粒子**：暗紅粒子從裂縫湧出 + 黑色煙霧 + 紅色閃電
- 閃電時卡片微震
- 卡名：暗紅脈動文字
- 圓角：6dp（更小 = 更銳利 = 危險感）
- 光效面積：100%（反向光 — 暗底上的紅色能量）
- 整體感覺：在白色收藏冊中唯一的黑卡 → **Von Restorff 效應 = 視覺核彈**

## 十一、設計注意事項

### 核心原則
1. **照片永遠是主角** — 卡框是「裝裱」，不能搶過照片本身
2. **低品質也好看** — 凡品是乾淨的白卡，不是「醜版傳說」
3. **一眼辨識品質** — 靠邊框顏色 + 光效面積，3 秒內縮略圖可辨
4. **深色 UI + 白色卡片** — 卡片在深色背景上像發光的收藏品

### 一人團隊限制
- 所有卡片視覺都用程式碼生成（CustomPainter + Shader），不用 PNG
- UI 可以用 Figma 設計，但卡片視覺只需要做**靜態 Mockup** 當目標參考
- 不需要做動畫稿（動畫用程式碼實現）
- 需要的外部資源只有：3 個字體 + 8 個音效檔

### 社群分享最佳化
- 卡片分享圖：1080 × 1350 px（Instagram 最佳）
- 白底卡片在社群瀑布流中像「獨立物件」浮起
- 全息效果只能用影片展示 → TikTok/Reels 友好
